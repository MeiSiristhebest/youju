# AI 对话功能设计与问题检查计划

## 一、问题调研结论

根据用户反馈和代码审查，AI 对话功能存在以下问题，按严重程度排序：

### P0 - 核心功能故障

1. **AI 只返回引用编号，没有实际回答内容**
   - 现象：用户提问后，AI 消息区域只显示 `[1] [2] ... [20]` 等引用编号，没有文字回答
   - 可能原因：
     - 系统提示词要求 AI 使用工具检索后回答，但模型没有正确执行工具调用后的回答步骤
     - `stopWhen: isStepCount(3)` 限制可能导致流程提前终止
     - 工具返回的检索结果格式可能与系统提示词预期不一致
   - 涉及文件：
     - `youju-server/src/ai/adapters/chatAdapter.ts` - 工具调用和流式处理
     - `youju-server/src/ai/prompts/versions/v1/chat.system.md` - 系统提示词

### P1 - 体验与一致性问题

2. **AI 对话与任务/场景绑定不明显**
   - 现象：用户说"AI 对话好像并没有针对这个任务和场景进行绑定，因为我不能选择指定的素材"
   - 实际情况：代码中已有场景绑定（`scenarioType`）和指定素材功能（`contextScope: 'custom'`），但 UI 入口不明显
   - 涉及文件：
     - `youju-app/src/components/chat/ChatInput.tsx` - 上下文范围选择器
     - `youju-app/src/components/chat/ChatPanel.tsx` - 对话面板

3. **系统提示词为英文**
   - 现象：`chat.system.md` 全部是英文
   - 问题：虽然系统提示词英文不影响输出（提示词要求中文回答），但增加维护成本，且可能导致某些模型对指令理解有偏差
   - 涉及文件：
     - `youju-server/src/ai/prompts/versions/v1/chat.system.md`

4. **对话场景化推荐问题缺失或不精准**
   - 现象：ChatEmpty 组件的推荐问题可能没有针对当前场景
   - 涉及文件：
     - `youju-app/src/components/chat/ChatEmpty.tsx`

### P2 - 优化与增强

5. **对话历史摘要与长期记忆体验不明显**
   - 已有 memoryService，但用户感知不到
   - 涉及文件：
     - `youju-server/src/domain/services/memoryService.ts`

6. **缺少对话与分析结果的联动**
   - 用户可能想在对话中追问某个风险点的细节，但需要手动描述
   - 可优化：从风险详情页点击"追问 AI"自动填入上下文

## 三、具体修改步骤

### 步骤 1：排查并修复 AI 只返回引用的问题（P0）

1. **分析 chatAdapter.ts 中的工具调用流程**
   - 检查 `streamText` 配置和 `stopWhen: isStepCount(3)` 是否正确
   - 检查 `onChunk` 回调是否正确处理 text-delta
   - 确认工具调用后 AI 是否继续生成回答

2. **增强系统提示词**
   - 明确要求：调用 `searchDocs` 工具后，必须基于检索结果生成完整回答
   - 添加工具调用后的回答格式强化说明
   - 中文化系统提示词

3. **验证修复**
   - 使用真实 API Key 测试对话功能
   - 确认返回内容包含实际回答和引用编号

### 步骤 2：强化场景与素材绑定的可见性（P1）

1. **ChatPanel 标题区优化**
   - 在副标题显示当前场景类型（如"租房签约审核"）
   - 显示当前关联的素材数量
   - 点击可查看/修改关联素材

2. **ChatInput 上下文选择器优化**
   - 增大选择器尺寸或添加引导提示
   - 在"指定素材"模式下显示更清晰的选中状态
   - 添加"仅分析结果相关素材"快捷选项

3. **ChatEmpty 场景化推荐**
   - 根据 `scenarioType` 显示不同的推荐问题
   - 租房场景：合同条款类、费用类、设施类等问题
   - 求职场景：薪资类、福利类、职责类等问题

### 步骤 3：对话与分析结果联动（P2）

1. **风险卡片添加"追问 AI"入口**
   - 点击后自动切换到 AI 对话标签
   - 自动填入风险相关的问题模板
   - 自动限定上下文为该风险相关的证据材料

2. **证据引用跳转**
   - 对话中引用的 [n] 点击可跳转到对应素材高亮
   - 已有 SourceDetailModal，需确保对话引用能正确触发

### 步骤 4：系统提示词中文化与优化（P1）

1. **翻译 chat.system.md 为中文**
   - 保持所有指令的准确性
   - 统一使用中文标识（如"结论"、"详细说明"、"建议"）

2. **优化提示词结构**
   - 强化"工具调用后必须回答"的指令
   - 添加回答质量检查清单
   - 优化引用格式说明

## 四、潜在依赖与注意事项

1. **Vercel AI SDK v7 版本兼容性**
   - `streamText` 和 `isStepCount` 的行为可能与版本相关
   - 需确认当前 SDK 版本的工具调用流程

2. **不同模型对工具调用的支持差异**
   - 不同 Provider（OpenAI、Anthropic、DeepSeek 等）工具调用行为可能不同
   - 需确保在多种模型下都能正常工作

3. **流式输出解析**
   - 前端 SSE 解析需要正确处理 token 事件
   - 需检查 `readSSEStream` 是否正确处理多事件类型

4. **RAG 检索质量**
   - 如果检索结果质量不高，AI 可能无法生成有效回答
   - 需确认 embedding 和 rerank 配置正确

## 五、风险与应对

| 风险 | 影响 | 应对方案 |
|------|------|---------|
| 工具调用后无回答是模型本身的问题 | 高 | 尝试不同模型验证；调整提示词强化指令；考虑使用多轮调用模式 |
| 中文化提示词导致效果下降 | 中 | 对比中英文提示词效果；保留英文版本作为备选 |
| 前端流式渲染逻辑有 bug | 中 | 添加详细日志；对比非流式结果验证 |
| RAG 检索结果为空导致拒答 | 中 | 优化检索策略；添加检索为空时的引导话术 |

## 六、验证标准

1. **核心功能验证**
   - AI 对话能返回完整的中文回答，包含引用标记
   - 引用 [n] 点击能跳转到对应素材
   - 流式输出正常，逐字显示

2. **场景绑定验证**
   - 不同场景的对话有对应的推荐问题
   - 对话标题显示当前场景
   - 指定素材功能可用且可见

3. **质量验证**
   - 回答基于检索内容，不胡说八道
   - 引用准确，每个事实都有来源
   - 回答结构清晰（结论→详情→建议）
